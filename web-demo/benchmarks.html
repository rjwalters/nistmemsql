Methodology

  • Hardware: GitHub Actions runners (ubuntu-latest, 2-core CPU)
  • Python: 3.11
  • Framework: pytest-benchmark v4.0.0
  • Iterations: Minimum 5 rounds with warmup enabled
  • Data: Deterministic test data for reproducibility
  • Comparison: Head-to-head on identical hardware

All benchmarks measure end-to-end query execution time including parsing, execution, and result fetching. Both databases run in-memory to eliminate disk I/O variance. Results are automatically updated on every commit to the main branch.
